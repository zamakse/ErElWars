# 에르엘워즈 (ErElWars) — CLAUDE.md

## 프로젝트 개요

에르엘워즈2(AREL WARS 2, 게임빌 2012)를 레퍼런스로 삼아 제작하는 2D 횡스크롤 공격형 라인 전략 게임.
플레이어가 마나(자원)를 소비해 병력을 생산하고, 자동 전투를 통해 상대 기지를 파괴하는 것이 목표.

- 장르: 공격형 라인 디펜스 (Attack-type Line Defense)
- 플랫폼: 모바일(Android/iOS) 또는 웹 (미정)
- 엔진 후보: Unity(C#) / Godot(GDScript)
- 레퍼런스: 에르엘워즈2 (AREL WARS 2, Gamevil, 2012)

## 원작(에르엘워즈2) 핵심 시스템 레퍼런스

### 전투 구조
- 좌측(플레이어) vs 우측(적 AI)
- 라인 구성: 지상 2라인 + 공중 1라인 (총 3라인)
- 병력 자동 전진 → 충돌 시 자동 공격 → 기지 HP 0이면 승패 결정
- 성 HP는 좌측 상단(플레이어) / 우측 상단(적)에 표시

### 종족 구성 (3종족)

| 종족 | 특성 | 영웅 | 플레이 스타일 |
|------|------|------|--------------|
| 데븐 | 악마 계열, 소환비용 높음, 범위공격 특화 | 아도니스 | 고비용 고효율, 광역 딜 |
| 버스터 | 수인 계열, 근접 유닛·영웅 성능 발군 | 플로라 | 공격형, 근접 돌격 |
| 유니언 | 전작 흄+엘븐 연합, 방어·공중 부대 강함 | 딘 | 방어형, 공중 병력 활용 |

### 영웅 시스템
- 영웅은 전장에 직접 출진 가능
- 영웅 사망 시 일정 시간 재출진 불가
- 레벨업 시 스탯 직접 분배: STR(힘), CON(체력), DEX(민첩), INT(지능)
- 영웅 스킬 슬롯: 4칸 (액티브 + 패시브 혼합)
- INT는 마나 최대치만 증가 → 과투자 주의

### 자원 시스템
- 마나 기반 자원 구조
- 시간 경과 자동 생성 + 업그레이드로 생산 속도 증가 가능
- 유닛 생산 및 스킬 사용 시 마나 소모
- 마나 게이지와 인구수는 화면 우측 하단 표시

### 유닛 시스템
- 유닛 슬롯: 8개 (초반 일부만 개방, 이후 해금)
- 아이템 슬롯: 최대 3개
- 유닛 속성: 비용(마나), HP, 공격력, 방어력, 이동속도, 공격속도, 사거리
- 상성: 딜러 → 기병에 약함 / 기병 → 탱커에 약함 / 탱커 → 딜러에 약함

### 강화·육성 시스템
- 강화: 골드 소비로 유닛 능력치 상승 (HP/공격/방어 중 1개씩)
- 광폭: 제물 유닛을 소비해 대상 유닛의 이동속도·공격속도 영구 상승
- 장비: 영웅 전용 장비 장착 및 강화 가능

### 스테이지 구성
- 총 5개 테마 × 15스테이지 = 75스테이지
- 노드 기반 월드맵, 순차 해금
- 일반 모드 클리어 → 하드 모드 해금

### UI 레이아웃 (원작 기준)
- 좌/우 상단: 아군/적 성 HP
- 우측 하단: 마나 게이지 + 인구수
- 하단: 유닛 생산 버튼 슬롯 (8칸) / 슬롯 전환 시 아이템·스킬 칸
- 중앙 상단: 일시정지 메뉴
- 영웅 상태창 + 간이 미니맵 (라인 전환 조작 포함)

## 코드 작성 규칙

- 함수·변수명은 영문 camelCase
- 클래스명은 PascalCase
- 주석은 한국어로 작성
- 매직 넘버 사용 금지 → 상수 또는 데이터 파일로 분리
- 유닛·영웅 수치는 JSON으로만 관리, 코드에 하드코딩 금지
- 라인(지상/공중) 구분은 enum LineType { Ground1, Ground2, Air } 사용
- 전투 로직과 UI 레이어 반드시 분리

## 개발 우선순위

1. 단일 라인 기본 전투 (이동·충돌·공격·사망)
2. 마나 자동생성 및 유닛 생산 로직
3. 적 AI 유닛 생성
4. 다중 라인 (지상2 + 공중1) 확장
5. 영웅 시스템 (출진·스탯분배·스킬)
6. 강화·광폭·장비 시스템
7. UI 구성
8. 스테이지·월드맵 시스템
9. 종족 선택 화면
10. 밸런싱 및 최적화

### 새 유닛 추가
```

라인: 지상1 / 지상2 / 공중
속성:
  - 마나 비용:
  - HP:
  - 공격력:
  - 방어력:
  - 이동속도:
  - 공격속도: (초)
  - 사거리: (근접이면 생략)
```
